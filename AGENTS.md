# This project — Omnitrix Game

A **Pebble game** (not a watchface) themed on the Ben 10 Omnitrix: a full-screen
animated badge state machine driven by the buttons and internal timers (no real
battery/clock dependency). Almost all logic is in one file: `src/c/main.c`.

## Layout
```
src/c/main.c        # the whole app
package.json        # manifest (uuid, platforms, resources, watchface:false)
resources/images/menu_icon.png   # 25x25 launcher icon (menuIcon:true), baked in
store/              # marketing assets for the Rebble listing (NOT baked in)
build/              # waf output (.pbw etc.)
```

## Build / run / verify
```bash
# Primary loop — connects to the emulator started by the Pebble VS Code extension:
pebble build && pebble install --emulator emery --vnc

pebble build                                 # builds all targetPlatforms
pebble install --emulator emery

pebble screenshot --emulator emery out.png
pebble emu-button --emulator emery click <up|select|down|back>
pebble kill                                  # if "multiple emulators running"
```
The VS Code Pebble extension starts an `emery` emulator; use
`--emulator emery --vnc` to install into it. (Other platforms build fine too —
swap the codename for one-off checks, e.g. `basalt`, or round `chalk`/`gabbro`.)
Verify UI changes by **screenshotting the emulator**, not by reasoning alone.
`pebble kill` + reinstall to reset to idle. Round platforms (chalk/gabbro) boot
slower — retry the screenshot if it fails.

### Editor diagnostics are misleading
clang will flag `pebble.h not found`, `unknown type 'Window'`, etc. These are
**false** (the SDK include path isn't visible to clang). **`pebble build` is the
source of truth** — trust it, ignore the clang panel.

## Architecture — declarative node router
The game is a small router described as **data**. Edit tables, not scattered
`switch`es.

- **`NODES[]`** — one row per screen. Declares `build(BuildContext)` (render),
  `auto_frames`/`auto_to` (timed auto-advance; how timed journeys are expressed),
  `on_click(button)` (in-node action), `locked` (input only splashes red — the
  discharged state), `transient` (ignore all input).
- **`NAV[]`** — declarative edges: `from` node + `button` -> `to` node, optionally
  via a colour flash. Yarn-style "options lead to nodes."
- **Router** (`router_goto`/`router_click`/`router_tick`) composes cross-cutting
  rules on top: global long-press toggles (Hijacked/Master/DNA-Scan), the locked
  splash, transient skipping, per-node frame counter, BACK (edge or exit app),
  and a **vibration pulse on every transition** (`vibes_short_pulse` in
  `router_goto`).

## Rendering — Flutter-style widgets + shared template
- `BuildContext { ctx, rect }` passed to each node's `build`.
- Composable widgets: `Fill`, `Emblem`, `EnergyFrame`, `ScanBarV`, `GlitchScan`.
- **`BadgeScreen(Badge{fill, ring, emblem})`** is the reusable idle-mode template.
  **Every screen composes it** except `build_select` (the green diamond). Reuse
  it; don't hand-roll new field+emblem code.
- The emblem is a **90-degree-rotated (vertical) hourglass**, border ring on the
  left/right edges. Keep new art (incl. icons/banner) in this orientation.

## Timing is frame-based
Anim timer ticks every `ANIM_INTERVAL_MS` (120 ms). Durations use
`SECS_TO_FRAMES(s)` and count `s_node_frame` (reset on each `router_goto`). No
wall-clock dependency; ~120 ms accuracy.

## Adding a screen (common task)
1. Write `static void build_x(BuildContext c)` — usually `BadgeScreen(...)` + an
   overlay widget.
2. Add a `NODES[NODE_X] = { ... }` row.
3. Add `NAV[]` edges. No button-handler or dispatch changes needed.

## Conventions & gotchas
- Keep it in `src/c/main.c`; match the terse, tabular style.
- Base colours black/white; accents green/orange/red/yellow/purple/cyan. `flip()`
  is a global colour hook (currently passthrough).
- Vibration fires automatically on transitions via the router — don't sprinkle
  `vibes_*` in handlers (exception: the locked red splash).
- `store/` assets are store artwork, **not** Pebble `resources/media` (resource
  budget ~128 KB on aplite). Only `menu_icon.png` belongs in `resources/`.
- Asset PNGs are generated by pure-Python encoders (no deps); keep emblems in the
  vertical orientation.

## Publishing (Rebble)
- `watchapp.watchface` is `false`. Bump `version` in `package.json` per release.
- Upload the `.pbw` + `store/` assets manually at the Rebble developer portal
  (dev-portal.rebble.io / appstore-api.repebble.com/dashboard).
- Trademark note: "Omnitrix"/Ben 10 is Cartoon Network IP — a public listing with
  the name/artwork carries takedown risk.
